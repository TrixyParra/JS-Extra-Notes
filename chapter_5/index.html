Primitive and Reference Types

- Stored in two different paths of memory.

Primitive Types: stored in stack - different values in variable but space is limited, can be quicker. A change of value makes a copy of a new Primitive value on the Stack but doesn't change the other/previous one.
  • numbers
  • strings
  • booleans
  • null
  • undefined
  • symbols
Reference Types: stored in heap - more space available, holds bigger and more complex types (objects, arrays), can be slower too. Copies of Reference Type make the same reference back to the Object stored in the Heap resulting in making a copy of the Pointer in the Stack based on the data in the Heap.
  • all types of objects:
    • object literals
    • arrays
    • functions
    • dates
    • all other objects
